# 三套 PID 独立测试与调参教程

本文对应以下三套互相独立的测试程序：

| 测试 | 程序 | 参数文件 | 测试编号 |
| --- | --- | --- | --- |
| 八路灰度循迹 PID | `gray_pid_tracking_test.c` | `gray_pid_tracking_test.h` | 11 |
| 陀螺仪航向串级 PID | `gyro_pid_control_test.c` | `gyro_pid_control_test.h` | 12 |
| 双电机速度闭环 PID | `motor_speed_pid_test.c` | `motor_speed_pid_test.h` | 13 |

三套程序不会同时控制电机。烧录一次后，可直接在 OLED 菜单中选择其中一项运行。

## 一、开始前必须做的准备

### 1. 安全要求

1. 第一次运行任何电机测试时，先把车架空，使两个驱动轮离地。
2. 确认复位键或断电开关能够立即触及。
3. OLED 出现 3 秒倒计时之前，电机保持为 0。
4. 先用较低 PWM 完成方向检查，再让小车落地。
5. 调参时每次只改一个参数，并记录修改前后的曲线。
6. 不要在电机运行时插拔编码器、陀螺仪或电机线。

### 2. XTARK MC520P30 全工程统一配置

本工程的驱动电机统一为 `XTARK_MC520P30_12V` 的 13 线霍尔编码器版，型号参数集中放在 `custom_src/drivers/actuators/motor/motor_config.h`。速度环、普通电机测试和编码器测试均引用该文件；底层继续使用 TB6612 单 PWM 加双方向脚驱动，不恢复旧工程中的 DRV8870 双 PWM 接法。

| 项目 | 当前统一值 |
| --- | --- |
| 额定电压 | 12 V |
| 减速比 | 1:30 |
| 空载输出转速 | 360±20 RPM |
| 编码器单相线数 | 电机轴每圈 13 个方波周期 |
| 单相输出轴周期数 | 390 cycles/rev |
| 当前四倍频输出轴计数 | 1560 counts/rev |
| 额定扭矩 | 1.5 kg·cm |
| 编码器供电范围 | 3.3～5 V，本板建议使用 3.3 V |
| PWM 定时器 | 10 MHz、周期 3000、约 3.33 kHz |
| 软件 PWM 指令范围 | -3000～+3000 |

当前左右关系为：左电机 C/M1 使用 `motor_pwms[0]`，PWMA=PB15、IN1=PB7、IN2=PB6，编码器 A=PB13、B=PB12、索引 0；右电机 D/M2 使用 `motor_pwms[1]`，PWMB=PB16、IN1=PA12、IN2=PA13，编码器 A=PA29、B=PB26、索引 1。若实车正方向与软件定义相反，只修改 `MOTOR_LEFT_DIRECTION_INVERTED` 或 `MOTOR_RIGHT_DIRECTION_INVERTED`，不要在各个 PID 程序中分别交换正负号。

底层驱动会同时按照 `MOTOR_PWM_COMMAND_MAX` 和定时器实际周期限幅，避免错误参数写入超过 PWM 周期的比较值。普通正反转测试在换向前增加了 500 ms 滑行停顿，避免 MC520P30 在仍高速旋转时直接反接。

陀螺仪航向保持测试默认 `GYRO_PID_ALLOW_MOTOR_REVERSE=0`。当前基础 PWM 为 700、最大差速为 600，因此正常控制范围内左右轮命令约为 100～1300，不会为了修正航向突然反转。只有专门调试原地转向、且已经完成安全换向测试后，才可以把反转开关设为 1。速度环检测到 PWM 较高但编码器无脉冲时约 0.5 秒停机，减少堵转或编码器断线时持续通电的时间。

各电机测试当前使用的范围：

| 测试 | 基础/测试 PWM | 最大 PWM | 相对 3000 周期 |
| --- | ---: | ---: | ---: |
| 普通方向测试 | 1500 | 1500 | 50% |
| 编码器测试 | 900 | 900 | 30% |
| 灰度循迹 | 900 | 1800 | 30% / 60% |
| 陀螺仪直行 | 700 | 1400 | 23% / 47% |
| 速度环 | 由 PID 计算 | 1600 | 最大约 53% |

这些值是首次调试起点，不是最终性能参数。软件 PWM 限幅不能替代过流保护；必须确认电机电源接近 12 V，并核对 TB6612 驱动板的持续电流、峰值电流和散热能力。首次运行仍要架空车轮，确认方向、编码器符号和每圈 1560 计数后再落地。

### 3. 选择测试程序

复位后 OLED 显示 `TEST MENU`。短按 K3 向下移动，找到 `GRAY PID`、`GYRO PID` 或 `MOTOR SPEED PID`，再短按 K4 确认。

这三项都会先经过菜单统一的 2 秒电机安全倒计时，然后进入各自原有的初始化和倒计时流程。结束测试时按开发板复位键，即可回到菜单并选择另一项，无需修改宏、重新编译或重新烧录。

编码器每圈计数已经校准完成，速度测试会循环执行 `100 -> 50 -> 120 RPM` 阶跃。只有修改 PID 参数或程序代码后才需要在 Keil 中 Rebuild 并重新烧录。

## 二、调试串口接线与读取方法

三套 PID 测试统一使用独立的 `DEBUG_UART/UART3` 输出调参数据，不占用摄像头的 UART2、云台的 UART0，也不占用陀螺仪的 UART1：

| 项目 | 设置 |
| --- | --- |
| 单片机发送脚 | PA14（UART3 TX） |
| 单片机接收脚 | PA25（UART3 RX） |
| 波特率 | 115200 |
| 数据格式 | 8 数据位、无校验、1 停止位（8N1） |
| 行结束 | LF（`\n`） |
| VOFA+ 协议 | FireWater（纯数字、逗号分隔） |

接线：

```text
MSPM0 PA14  -> USB-TTL RX
MSPM0 PA25  <- USB-TTL TX
MSPM0 GND   -> USB-TTL GND
```

只使用 3.3 V TTL 电平。当前 PID 程序主动发送波形数据，PA25 同时已配置为 UART3 RX，后续可以增加串口命令来在线修改参数。摄像头使用 `UART2 PA21/PB18`，云台使用 `UART0 PA0/PA1`，陀螺仪使用 `UART1 PA8/PA9`，四路串口互不共用。

### 1. VOFA+ FireWater 设置

1. 让无线设备进入串口透传/串口调试状态；本项目只使用它的无线串口功能，不用它烧录程序。
2. 在 VOFA+ 中选择该无线串口对应的 COM 口。
3. 波特率设置为 115200，数据格式为 8N1。
4. 协议引擎选择 `FireWater`。
5. 添加波形显示控件并打开串口。
6. 固件每一行只发送固定数量的数字，不发送表头，也不发送 `GRAY/GYRO/SPD` 等文字。
7. VOFA+ 中的通道按 `CH0、CH1、CH2...` 顺序出现，再按照本文每个测试的通道表重命名。

如果 VOFA+ 只能看到原始字符但没有波形，优先检查协议是否选成 FireWater、波特率是否为 115200，以及每行是否以换行结束。不要在固件波形流中加入 `printf("PID start")` 一类文字。

当前采样输出频率：

| 测试 | 控制周期 | VOFA+ 输出频率 |
| --- | --- | --- |
| 灰度循迹 | 10 ms | 约 20 Hz |
| 陀螺仪航向 | 10 ms | 约 20 Hz |
| 电机速度环 | 20 ms | 约 25 Hz |

波形发送频率低于控制频率，是为了减少阻塞式串口发送对 PID 周期的扰动。无线链路可能把若干行数据成批送到电脑，因此横轴优先按采样点理解，不要把电脑接收时刻当成精确的 MCU 控制周期。

### 2. XDS110 有线烧录 + 无线串口的推荐工作流程

程序通过 `XDS110` 的 SWD 接口有线下载和调试；无线设备只连接 `UART3 PA14/PA25`，负责把 PID 数据透传给 VOFA+。两条链路相互独立，因此使用 XDS110 烧录时通常不需要关闭 VOFA+，也不存在为了烧录而释放无线串口 COM 的问题。

推荐按以下顺序调参：

1. 保持无线串口模块与 PA14、PA25、GND 的连接，并在 VOFA+ 中打开 FireWater 波形。
2. 每次只修改一个 PID 参数，然后在 Keil 中执行 Rebuild。
3. 使用 XDS110 通过 SWD 下载程序，并让单片机复位运行。
4. 下载和复位期间 VOFA+ 波形会短暂停顿或断开一小段，这是正常现象；无线串口 COM 可以继续保持打开。
5. 等待程序重新初始化和 OLED 倒计时结束，清空 VOFA+ 历史波形或开始一段新记录。
6. 执行一次幅度和持续时间尽量一致的标准扰动，保存波形截图或数据，并记录本次参数。
7. 重复以上过程，对比回正时间、超调、振荡和输出饱和情况。

只有无线串口驱动异常、COM 无法继续接收时，才需要在 VOFA+ 中关闭并重新打开串口。若使用 Keil 在线调试并让 CPU 停在断点，PID 控制和 UART 发送都会同时暂停，此时的断流不能用来评价控制效果。

小车由电池供电时，无线串口模块通常只连接 TX、RX、GND，不要再用它的 VCC 给整车反向供电。XDS110、无线串口模块和小车控制板必须共地，UART 电平必须是 3.3 V TTL。XDS110 复位目标板时，VOFA+ 偶尔丢弃一行不完整的 FireWater 数据也属正常，复位后固定格式的波形会自动恢复。

### 3. 波形调参统一判据

每组参数至少观察以下指标：

| 指标 | 波形含义 |
| --- | --- |
| 上升/回正时间 | 从施加目标或扰动到第一次接近目标的时间 |
| 最大超调 | 实测曲线越过目标的最大幅度 |
| 稳定时间 | 进入目标附近允许误差带后不再离开的时间 |
| 稳态误差 | 稳定后目标与反馈之间的平均差值 |
| 振荡 | 误差是否持续正负交替，以及振幅是否衰减 |
| 输出饱和 | PWM 或修正量是否长时间贴着限幅 |

调参时把“目标与反馈”放在同一张图，把“误差”放在第二张图，把“PWM/控制输出”放在第三张图。先保存一组基准曲线，每次只修改一个参数，并使用相同目标、相同扰动幅度和相同路面重复测试。

名称中带 `_x10` 的字段是放大 10 倍后的整数。例如 `yaw_x10=-123` 代表 `-12.3°`，`left_x10=985` 代表 `98.5 RPM`。这样可以避免嵌入式浮点格式化给控制周期带来额外负担。

## 三、灰度循迹 PID 调试

### 1. 测试目的和控制结构

程序把八路传感器按车体物理位置从左到右排列。显示的 8 位中，最左字符对应最左探头，最右字符对应最右探头。

有效探头位置被映射为：

```text
左侧                                        右侧
 -7   -5   -3   -1   +1   +3   +5   +7
```

多个探头同时检测到黑线时取加权平均。该位置直接作为 PID 误差：

```text
correction = Kp * error + Ki * integral + Kd * derivative
left_pwm   = base_pwm + correction
right_pwm  = base_pwm - correction
```

程序丢线后会按最后一次误差方向短时间搜索；超过设定周期仍找不到线，就停止电机。

### 2. 串口数据

VOFA+ FireWater 每行 6 个纯数字通道：

```text
sensors_value,error_x10,correction_pwm,left_pwm,right_pwm,line_found
```

| VOFA+ 通道 | 建议名称 | 含义 |
| --- | --- | --- |
| CH0 | `sensors_value` | 八路传感器位图的十进制值，可转换为 8 位二进制查看 |
| CH1 | `error_x10` | 位置误差乘 10，负数在左侧，正数在右侧 |
| CH2 | `correction_pwm` | PID 输出的差速修正量 |
| CH3 | `left_pwm` | 左轮实际 PWM |
| CH4 | `right_pwm` | 右轮实际 PWM |
| CH5 | `line_found` | 1 表示检测到线，0 表示丢线 |

主要画三条曲线：`error_x10`、`correction_pwm`，以及左右 PWM。理想状态是误差回到 0 附近，修正量不会持续来回打满。

### 3. 第一步：只检查传感器和方向

1. 先选择普通灰度测试 `MODULE_TEST_GRAY_SENSOR`，不要让车运动。
2. 从左到右逐个用黑线遮挡探头。
3. OLED 应按物理顺序从左到右显示对应的 `1`。
4. 回到灰度 PID 测试，架空车轮。
5. 把黑线放到传感器左侧，确认 `error_x10` 为负；放到右侧应为正。
6. 让黑线偏向一侧，确认左右轮差速会把车头修正回线中央。如果越修越偏，需要交换差速方向，而不是继续增大 PID。

### 4. 第二步：调 Kp

在 `gray_pid_tracking_test.h` 中先设置：

```c
GRAY_PID_KI = 0
GRAY_PID_KD = 0
```

把 `GRAY_PID_BASE_PWM` 降到车辆能够稳定运动的较低值。调节方法：

1. 从较小 `GRAY_PID_KP` 开始。
2. 每次增加约 10%～20%，观察弯道修正和直线摆动。
3. Kp 太小：进入弯道反应慢，误差长时间偏在一侧，容易冲出线。
4. Kp 太大：直线上左右高频摆动，`error_x10` 正负交替，左右 PWM 差值过大。
5. 找到刚开始持续摆动的 Kp 后，回退约 20%～30%。

### 5. 第三步：调 Kd

保持 Kp 不变，从较小的 `GRAY_PID_KD` 开始增加：

1. Kd 合适：相同 Kp 下摆动更快衰减，入弯后不会反复穿过中心线。
2. Kd 太小：回正后仍左右来回摆。
3. Kd 太大：PWM 对传感器跳变非常敏感，输出尖峰明显，电机声音发抖。
4. `GRAY_PID_D_FILTER_KEEP` 越大，微分越平滑但响应越慢；通常先保持默认值。

### 6. 最后再加少量 Ki

灰度循迹通常不需要很大的积分。只有车辆在直线或固定方向弯道存在长期偏差时，才逐步增加 `GRAY_PID_KI`。

Ki 太大时会出现明显的慢周期左右摆动，并且离开弯道后仍保留错误修正。此时应减小 Ki 或减小 `GRAY_PID_INTEGRAL_LIMIT`。

### 7. 其他重要参数

| 参数 | 作用 |
| --- | --- |
| `GRAY_PID_BASE_PWM` | 直线基础速度 |
| `GRAY_PID_MAX_PWM` | 单轮最大 PWM |
| `GRAY_PID_OUTPUT_LIMIT` | 最大差速修正量 |
| `GRAY_PID_CURVE_SLOWDOWN_PER_UNIT` | 误差越大时主动减速的强度 |
| `GRAY_PID_LOST_SEARCH_CYCLES` | 丢线后允许搜索的时长 |

先在低速调好 PID，再逐渐提高基础速度。提高速度后通常需要重新增加 Kd，并检查弯道减速量。

## 四、陀螺仪目标航向保持双环 PID 调试

### 1. 本测试要验证的功能

本测试只验证陀螺仪目标航向闭环，不使用灰度传感器，也不使用编码器反馈。目标功能是：

1. 自动扫描陀螺仪串口波特率并确认存在有效数据。
2. 启动前发送航向清零命令，清零时车头方向定义为 `0°`。
3. 读取一个可配置的目标航向角 `target_yaw`。
4. 小车以固定基础 PWM 持续向前行驶。
5. 当前航向偏离目标后，控制器产生左右轮差速，使车头回到目标航向。
6. 回到目标后差速自动减小，小车继续沿目标航向前进。
7. 目标航向以后即使由其他程序动态修改，同一套控制器也能自动转向新目标并继续保持。

该测试验证的是“航向保持”，不是位置闭环。陀螺仪能够检测车头转角，但不能检测小车在保持相同朝向时发生的整体横向平移。

连续约 300 ms 收不到有效陀螺仪帧时，程序把两个电机 PWM 置 0、清除 PID 积分并熄灭状态灯。数据恢复后控制器从清空的积分状态重新工作。

### 2. 双环结构和计算公式

外环使用航向角 PI，默认先关闭积分，只使用 P：

```text
angle_error = wrap180(target_yaw - yaw)

target_rate =
    angle_kp * angle_error
  + angle_ki * angle_integral
```

`angle_error` 始终归一化到 `-180°...+180°`，因此从 `+179°` 转向 `-179°` 时会选择只转 `2°` 的短路径，而不会错误地绕行 `358°`。

外环输出是目标 Z 轴角速度，并限制在：

```text
-GYRO_PID_TARGET_RATE_LIMIT_DPS
```

内环使用 Z 轴角速度 PI：

```text
rate_error = target_rate - z_rate

turn_pwm =
    rate_kp * rate_error
  + rate_ki * rate_integral
```

最后将内环输出作为左右轮差速：

```text
left_pwm  = base_pwm - turn_pwm
right_pwm = base_pwm + turn_pwm
```

程序按 `10 ms` 固定周期运行，两个积分都按照真实秒单位计算：

```text
integral += error * 0.01
```

目标角速度和差速 PWM 饱和时使用条件积分抗饱和：如果继续积分只会让输出在错误方向上越积越大，就暂停该方向的积分；误差开始帮助控制器退出饱和时允许积分释放。修改控制周期后，积分增益不会像旧的“每周期直接累加误差”写法那样按周期倍数突变。

角度外环不使用 D，因为陀螺仪 `GyroZ` 已经提供直接的角速度反馈；角速度环也不使用 D，避免放大角速度测量噪声。初始调试推荐的实际结构是：

```text
角度外环 P + 角速度内环 PI
```

### 3. 代码中的保守初始参数

当前首轮调试参数是：

| 参数 | 初值 | 单位/含义 |
| --- | ---: | --- |
| `GYRO_PID_ANGLE_KP` | 3.0 | `(°/s)/°`，每 1° 航向误差产生 3°/s 目标角速度 |
| `GYRO_PID_ANGLE_KI` | 0.0 | 初次调试关闭角度积分 |
| `GYRO_PID_ANGLE_INTEGRAL_LIMIT_DEG_S` | 20.0 | 角度积分状态限幅，单位 `°·s` |
| `GYRO_PID_RATE_KP` | 3.0 | `PWM/(°/s)` |
| `GYRO_PID_RATE_KI` | 12.0 | `PWM/°`，因为角速度误差积分后的单位为 `°` |
| `GYRO_PID_RATE_INTEGRAL_LIMIT_DEG` | 15.0 | 角速度积分状态限幅；默认最多贡献约 180 PWM |
| `GYRO_PID_TARGET_RATE_LIMIT_DPS` | 60.0 | 最大目标转向角速度 `°/s` |
| `GYRO_PID_TURN_PWM_LIMIT` | 600 | 最大左右轮差速修正量 |
| `GYRO_PID_BASE_PWM` | 700 | 两轮向前行驶的公共基础 PWM |
| `GYRO_PID_MAX_PWM` | 1400 | 本测试允许的单轮最大 PWM |

这组数值是根据以下条件得到的保守起点：

1. 电机是 `XTARK MC520P30 12V、1:30`，空载输出轴转速约 `360 RPM`。
2. 本工程 PWM 周期为 3000，基础 PWM 700 约为满量程的 23%。
3. 原厂示例换算到本工程后的角速度比例增益约为 `0.45 * 6 = 2.7 PWM/(°/s)`，因此首轮取便于计算的 `3.0`。
4. 原程序 `10 ms` 周期下的角速度积分等效增益约为 `0.025 * 6 / 0.01 = 15 PWM/°`；为降低首轮积分振荡风险，当前取 `12.0`。
5. 原角度 Kp 为 6.55，首轮降到 3.0，使 10° 偏差只要求约 30°/s，30° 以上逐步碰到 60°/s 限幅。
6. 基础 PWM 700、差速限幅 600 时，控制器正常输出范围为左/右轮约 `100...1300`，低于 1400 上限并且不会反转。

电机型号只能确定供电、减速比和大致安全 PWM，不能单独计算出实车最终 PID。最终参数还取决于车轮直径、左右轮距、整车质量、轮胎和地面摩擦、电池带载电压、电机死区及陀螺仪实际输出频率。

当前没有给 `turn_pwm` 增加固定死区。小车已经以 700 PWM 向前运动时，两台电机都不是静止起步状态；如果每次出现极小误差都额外跳变 96 PWM，反而容易在目标角附近左右抖动。

### 4. 设置目标航向角

陀螺仪清零后，车头初始方向是 `0°`。独立测试默认目标由头文件设置：

```c
#define GYRO_PID_INITIAL_TARGET_YAW_DEG 0.0f
```

常用测试值：

```c
0.0f     // 保持启动方向
30.0f    // 启动后转向 +30° 并保持
-45.0f   // 启动后转向 -45° 并保持
90.0f    // 启动后转向 +90° 并保持
```

后续应用程序需要运行中改变目标时，调用：

```c
gyro_pid_set_target_yaw(30.0f);
gyro_pid_set_target_yaw(-45.0f);
```

目标会自动归一化到 `-180°...+180°`。控制器发现目标发生明显变化时会清空旧目标积累的角度和角速度积分，防止上一段航向的积分把车辆继续推向错误方向。读取当前请求目标可调用：

```c
float target = gyro_pid_get_target_yaw();
```

独立测试当前没有实现串口命令解析；修改头文件目标值后重新编译即可测试不同目标。上面的运行时接口用于以后接入按键、上位机命令或任务状态机。

### 5. OLED 和串口数据

OLED 第一行数据显示当前角度 `Y` 和目标角度 `T`，第二行显示实际角速度 `W` 和目标角速度 `R`，第三行显示左右轮实际 PWM。

VOFA+ FireWater 每行输出 9 个纯数字通道：

```text
yaw_x10,target_yaw_x10,z_rate_x10,target_rate_x10,angle_error_x10,turn_pwm,left_pwm,right_pwm,stale_cycles
```

| VOFA+ 通道 | 建议名称 | 含义 |
| --- | --- | --- |
| CH0 | `yaw_x10` | 当前航向角乘 10 |
| CH1 | `target_yaw_x10` | 当前运行时目标航向角乘 10 |
| CH2 | `z_rate_x10` | 实测 Z 轴角速度乘 10 |
| CH3 | `target_rate_x10` | 角度外环给出的目标角速度乘 10 |
| CH4 | `angle_error_x10` | `wrap180(target-yaw)` 后的误差乘 10 |
| CH5 | `turn_pwm` | 角速度内环输出的转向差速量 |
| CH6 | `left_pwm` | 左轮实际 PWM |
| CH7 | `right_pwm` | 右轮实际 PWM |
| CH8 | `stale_cycles` | 连续没有新陀螺仪帧的 10 ms 周期数 |

调内环时，把 `target_rate_x10` 和 `z_rate_x10` 放在同一张图；调外环时，把 `target_yaw_x10` 和 `yaw_x10` 放在同一张图；同时单独观察 `turn_pwm` 是否长时间贴住 `+-600`。

### 6. 上电前检查与转向符号确认

第一次运行必须先架空驱动轮：

1. 确认陀螺仪与车体固定牢固，Z 轴与车辆偏航旋转轴一致。
2. 保持车辆完全静止，完成清零和 3 秒倒计时。
3. 静止时 `GyroZ` 应接近 0；如果长期偏离 0 超过约 `1～2°/s`，先执行零偏校准，不要用 PID 积分掩盖传感器错误。
4. 手动逆时针缓慢转动车体，记录 `Yaw` 和 `GyroZ` 的正负；两者在同一转动方向下必须同号。
5. 倒计时后两个轮子都应按车辆定义的前进方向旋转。
6. 手动把车头偏离目标约 5°，观察两轮差速是否试图让角度误差减小。
7. 如果差速会让误差继续增大，只把 `GYRO_PID_TURN_DIRECTION` 从 `1.0f` 改成 `-1.0f`，不要通过交换 Kp 正负或增大增益修复方向错误。
8. 确认最大差速下 PWM 仍在预期范围内，再让车辆落地。

架空状态只能检查电机和控制符号，不能完成动态调参。架空后车轮旋转不会带动车体产生真实角速度，因此内外环最终参数必须在有足够安全空间的地面上调节。

### 7. 第一步：先调角速度内环 PI

角速度内环决定“外环要求车头以某个速度转动时，车辆能不能及时做到”。调内环时暂时使用：

```c
GYRO_PID_ANGLE_KP = 1.0f
GYRO_PID_ANGLE_KI = 0.0f
GYRO_PID_RATE_KI  = 0.0f
GYRO_PID_TARGET_RATE_LIMIT_DPS = 30.0f
```

然后按以下步骤调 `GYRO_PID_RATE_KP`：

1. 把目标航向设为 0°，让小车低速向前行驶。
2. 从侧面短暂推动车头，制造约 5°～15° 航向误差后立即松手。
3. 在 VOFA+ 中比较 `target_rate_x10` 和 `z_rate_x10`。
4. `RATE_KP` 可从 `2.0` 开始，每次增加 `0.5` 左右。
5. Kp 太小：`z_rate` 明显小于目标，车辆转回来很慢，松手后长时间没有足够差速。
6. Kp 合适：`z_rate` 很快朝目标变化，方向一致，且不会持续正负翻转。
7. Kp 太大：`z_rate` 高频正负交替、左右 PWM 快速抖动，电机声音明显发颤。
8. 找到刚开始出现持续振荡的 Kp 后，回退约 20%～30%。

确定 Rate Kp 后加入 Rate Ki：

1. 从 `GYRO_PID_RATE_KI=0` 开始，每次增加约 `2.0`。
2. 相同扰动下，如果 `target_rate` 与 `z_rate` 长时间存在同方向差值，可增加 Ki。
3. Ki 合适：持续偏差逐渐消失，车辆受到稳定侧向扰动后仍有足够修正能力。
4. Ki 太大：松开扰动后仍保留很大差速，经过目标角后慢速来回摆动。
5. `GYRO_PID_RATE_INTEGRAL_LIMIT_DEG=15` 时，默认 Ki 12 最多贡献约 `12 * 15 = 180 PWM`；不要为了增加转向能力直接无限放大积分限幅。

首轮预计可用范围通常在：

```text
RATE_KP：2.0～6.0
RATE_KI：5.0～20.0
```

这只是当前车和 PWM 范围的搜索区间，不是保证值。如果 `target_rate` 符号合理而 `z_rate` 始终反向，先修正 `GYRO_PID_TURN_DIRECTION`；如果 `turn_pwm` 已长期达到 600 而转向仍明显不足，应检查基础 PWM、轮胎打滑、供电和机械结构，而不是继续加积分。

### 8. 第二步：再调航向角外环

内环能够跟随 `target_rate` 后，保持：

```c
GYRO_PID_ANGLE_KI = 0.0f
```

开始调 `GYRO_PID_ANGLE_KP`：

1. 把 `ANGLE_KP` 先设为 `1.5`，目标角速度限幅恢复为 `60°/s`。
2. 目标航向设为 0°，分别施加约 5°、10°、20° 的短暂扰动。
3. 每次只修改 Angle Kp，建议每次增加 `0.5`。
4. Angle Kp 太小：`target_rate` 本身就很小，即使内环跟随很好，Yaw 仍回正缓慢。
5. Angle Kp 合适：偏差越大回转越快，接近目标时目标角速度自然减小，Yaw 只有很小超调。
6. Angle Kp 太大：目标角速度快速打到限幅，Yaw 穿过目标后反复摆动，但 `z_rate` 可能仍能正确跟随目标。
7. 找到刚开始明显多次超调的 Kp 后，回退约 20%。

首轮预计 Angle Kp 通常落在：

```text
1.5～5.0 (°/s)/°
```

默认初值 3.0 表示：

```text
1°误差  -> 3°/s目标角速度
10°误差 -> 30°/s目标角速度
20°误差 -> 60°/s，刚好达到限幅
```

角度积分默认保持 0。只有满足以下全部条件时才考虑增加：

1. 角速度内环已经调稳。
2. `GyroZ` 零偏正常。
3. 小车在目标附近长期保留同方向的小角度误差。
4. `turn_pwm` 没有长期饱和。

此时可从 `ANGLE_KI=0.02` 开始，每次增加约 `0.02`，一般不要在首轮超过 `0.10`。Angle Ki 太大会出现慢周期蛇形摆动。角度外环不增加 D；需要更多阻尼时，优先检查角速度内环。

### 9. 第三步：目标角和抗扰测试

两环分别调稳后，按下面顺序验证完整功能：

1. `target_yaw=0°`：让小车正常前进，多次从左右两侧短暂推动车头，松手后应回到 0° 并继续前进。
2. `target_yaw=+30°`：重新编译运行，小车应向正角方向转到约 30°，随后沿 30° 航向持续行驶。
3. `target_yaw=-30°`：验证负方向的响应是否与正方向基本对称。
4. `target_yaw=+90°`：检查大角度误差时目标角速度是否停在 60°/s 限幅，接近目标后能否平滑减速。
5. 在已经稳定的目标航向上持续施加轻微转向阻力，然后松手，确认积分能够提供修正且不会在松手后长时间残留。
6. 分别在电池电量较高和较低时重复测试，记录参数对电压变化的敏感程度。

建议的首轮合格判据：

```text
静止 GyroZ 零偏：尽量小于 1～2°/s
小扰动后的稳态航向误差：小于约 1～2°
10°扰动：能明显回正且不出现持续摆动
目标角附近：turn_pwm 不持续正负高频翻转
大角度过程：turn_pwm 可以短时限幅，但不能回到目标后仍长时间饱和
数据状态：stale_cycles 经常回到 0，不能持续上升到 30
```

### 10. 根据曲线定位问题

| 现象 | 优先判断和处理 |
| --- | --- |
| `target_rate` 已合理，但 `z_rate` 幅值明显跟不上 | 增加 Rate Kp；检查转向能力、供电和机械摩擦 |
| `z_rate` 初期能跟随，随后长期差一点 | 小量增加 Rate Ki |
| `z_rate` 高频正负跳变，电机发抖 | Rate Kp 太大；确认没有错误固定死区 |
| `z_rate` 跟随很好，但 Yaw 回正很慢 | Angle Kp 太小 |
| Yaw 反复穿越目标，`target_rate` 也反复变号 | Angle Kp 太大，或 Rate 内环响应太慢 |
| 回到目标后仍长期保留差速 | Rate Ki 太大或积分限幅太大 |
| 小误差时左右轮突然来回跳变 | 检查是否自行恢复了固定差速死区；检查陀螺仪噪声 |
| 正目标角总向负方向转 | `GYRO_PID_TURN_DIRECTION` 符号错误，或 Yaw/GyroZ 正方向不一致 |
| 目标为 0 但 Yaw 持续单向漂移 | 重新做陀螺仪零偏校准并检查安装振动 |
| `turn_pwm` 长期贴住 `+-600` | 目标角速度过高、动力不足或控制方向错误；不要继续放大积分 |
| `stale_cycles` 上升到 30 | 检查 PA9、供电、共地、陀螺仪波特率及 UART 数据帧 |

如果需要把首轮参数进一步缩小到更准确的范围，请记录并提供：轮胎直径、左右轮中心距、整车质量、实际电池带载电压、车辆落地后刚能稳定前进的最小 PWM、陀螺仪每秒有效帧数，以及一次 `target_yaw/yaw/target_rate/z_rate/turn_pwm` 的 VOFA+ 数据。仅凭电机型号无法替代这些整车动态数据。

## 五、双电机速度闭环 PID 调试

### 1. 测试目的和反馈单位

左右轮各有一个独立 PI，并在闭环输出前加入目标转速前馈：

```text
目标 RPM * 前馈系数
        +
(目标 RPM - 实测 RPM) -> 单轮 PI
        =
单轮 PWM
```

当前测试只做正向速度闭环，PID 输出限制在 `0 ... MOTOR_SPEED_PID_MAX_PWM`，不会为了制动突然反转电机。

当前左右轮参数均为 `Kp=6.0、Ki=25.0、Kd=0.0`。前馈系数
`MOTOR_SPEED_PID_FEEDFORWARD_PWM_PER_RPM=8.0`，是根据 PWM 满量程 3000、
电机 12 V 空载约 360 RPM 得到的保守起点：

```text
3000 / 360 = 8.33 PWM/RPM
```

实际取 8.0，目标 100 RPM 时控制器一开始就给出约 800 PWM 的基础驱动力，
再由 PI 补偿负载和左右电机差异。这样不需要等待积分从 0 慢慢累积，上电加速和
目标阶跃响应会明显快于纯 PI。

当前编码器配置关系是：

```text
编码器 0 -> 左轮 C/M1 -> motor_pwms[0]
编码器 1 -> 右轮 D/M2 -> motor_pwms[1]
```

当前配置对应 `XTARK_MC520P30_12V` 的 13 线霍尔编码器版：额定电压 12 V、减速比 1:30、空载输出转速 360±20 RPM、额定扭矩 1.5 kg·cm。编码器供电范围为 3.3～5 V，与 MSPM0 直连时使用 3.3 V。编码器是 AB 相正交霍尔编码器，每个电机轴转一圈、每相输出 13 个方波周期，所以经过 1:30 减速箱后，单相对应输出轴每圈 `13 * 30 = 390` 个方波周期。

当前工程同时对 A、B 两相的上升沿和下降沿计数，属于四倍频译码，因此程序使用的输出轴每圈计数为：

```text
13 * 30 * 4 = 1560 counts/rev
```

因此统一配置 `MOTOR_ENCODER_COUNTS_PER_OUTPUT_REV` 为 1560，速度环的 `MOTOR_SPEED_PID_ENCODER_COUNTS_PER_REV` 直接引用该值。13 PPR 是电机轴单相信号周期数，不能在当前四倍频程序中直接把 13 或 390 当作最终 count/rev，否则 RPM 计算会偏大。如果以后改成只统计 A 相单边沿，才应使用 390。

360 RPM 是 12 V 空载参考值，不是建议直接使用的首次闭环目标。带轮胎、车重和路面负载后最高稳定转速会更低；首次调试仍建议从 40 RPM 架空检查，再在 60～100 RPM 区间开始整定。若实测每圈计数与 1560 明显不一致，必须先按后文方法实测校准，再调 PID。

### 2. 串口数据

VOFA+ FireWater 每行 13 个纯数字通道：

```text
target_x10,left_x10,right_x10,left_pwm,right_pwm,left_err_x10,right_err_x10,left_count,right_count,left_delta,right_delta,dt_ms,status
```

| VOFA+ 通道 | 建议名称 | 含义 |
| --- | --- | --- |
| CH0 | `target_x10` | 左右轮共同目标 RPM 乘 10 |
| CH1 | `left_x10` | 滤波后的左轮实测 RPM 乘 10 |
| CH2 | `right_x10` | 滤波后的右轮实测 RPM 乘 10 |
| CH3 | `left_pwm` | 左轮速度 PID 输出 |
| CH4 | `right_pwm` | 右轮速度 PID 输出 |
| CH5 | `left_err_x10` | 左轮速度误差乘 10 |
| CH6 | `right_err_x10` | 右轮速度误差乘 10 |
| CH7 | `left_count` | 左轮累计编码器计数 |
| CH8 | `right_count` | 右轮累计编码器计数 |
| CH9 | `left_delta` | 左轮本控制周期计数增量 |
| CH10 | `right_delta` | 右轮本控制周期计数增量 |
| CH11 | `dt_ms` | 本次实际控制间隔，通常约 20 ms |
| CH12 | `status` | 0 正常，-1 编码器方向错误，-2 无编码器脉冲 |

主要画 `target_x10`、`left_x10`、`right_x10` 三条速度曲线，同时观察 PWM 是否长期顶到上限。

编码器每圈计数校准完成后，OLED 恢复使用 6×10 字体显示四行：

```text
MOTOR SPEED PID
L:<left_rpm>
R:<right_rpm>
T:<target> P:<left>,<right>
```

`T` 是当前目标 RPM，`P` 后面依次是左右轮 PWM。需要再次校准累计计数时，
可临时恢复计数显示或使用串口输出的 `left_count/right_count`。

当前响应测试配置为：

```text
启动后 0～5 s：目标 100 RPM
       5～10 s：目标  50 RPM
      10～15 s：目标 120 RPM
随后从 100 RPM 重新循环
```

每段时间由 `MOTOR_SPEED_PID_STEP_HOLD_MS` 设置。把
`MOTOR_SPEED_PID_ENABLE_TARGET_PROFILE` 改为 `0` 后，程序不再循环切换，
而是始终使用 `MOTOR_SPEED_PID_TARGET_RPM`。

### 3. 先校验左右对应关系、方向和每圈计数

第一次测试必须架空车轮：

1. 先把 `MOTOR_SPEED_PID_ENABLE_TARGET_PROFILE` 设为 `0`，再把 `MOTOR_SPEED_PID_TARGET_RPM` 临时设低，例如 40 RPM。
2. 运行后确认左轮转动时 `left_count` 主要变化，右轮转动时 `right_count` 主要变化。
3. 两轮正转时 `left_x10`、`right_x10` 都应为正数。
4. 如果某一轮正转却报告负 RPM，程序会显示 `ENCODER SIGN ERROR` 并停止。在 `motor_config.h` 中翻转对应的 `MOTOR_LEFT_ENCODER_INVERTED` 或 `MOTOR_RIGHT_ENCODER_INVERTED`；不要修改 `MOTOR_SPEED_PID_*_ENCODER_SIGN` 的条件表达式。
5. 如果高 PWM 约 0.5 秒仍没有脉冲，程序显示 `ENCODER NO PULSE` 并停止。检查编码器供电、A/B 相引脚和 GPIO 中断；排除编码器问题前不要反复复位运行。
6. 校准每圈计数时，把目标曲线关闭并将 `MOTOR_SPEED_PID_TARGET_RPM` 临时设为 `0.0f`，通过串口的 `left_count/right_count` 或临时计数显示读取累计值；按车辆定义的正方向手动转动车轮整整一圈，`abs(新计数 - 旧计数)` 应接近 1560。

若左右映射错误，修改两个 `ENCODER_INDEX`；若每圈计数错误，先改 counts/rev，再开始调 PID。

### 4. 只调 Kp

先把左右 `KI`、`KD` 全部设为 0，并让左右 Kp 使用相同的较小初值：

1. 设定一个中低目标速度，例如 60～100 RPM。
2. 逐步增加 Kp，每次增加约 10%～20%。
3. Kp 太小：上升慢，实际速度长期明显低于目标。
4. Kp 合适：速度能较快接近目标，扰动后能恢复，没有持续振荡。
5. Kp 太大：速度在目标上下反复跳动，PWM 大幅来回变化，电机声音不稳定。
6. 找到刚开始持续振荡的 Kp 后回退约 20%～30%。

左右轮机械差异较大时，可以分别调 `LEFT_KP` 与 `RIGHT_KP`，不要用错误的编码器符号去补偿机械差异。

### 5. 加 Ki 消除稳态误差

保持 Kp 不变，从小到大增加左右 Ki：

1. Ki 的作用是补偿摩擦、电池电压和负载造成的长期速度误差。
2. Ki 太小：负载增加后速度持续低于目标。
3. Ki 合适：稳态速度逐渐贴近目标，负载变化后能恢复。
4. Ki 太大：速度发生慢周期上下摆动，超调后很久才能回来。
5. PWM 长期饱和时不要继续增加 Ki，应检查目标是否超出电机能力。
6. `MOTOR_SPEED_PID_INTEGRAL_OUTPUT_LIMIT` 限制积分项能贡献的最大 PWM。当前由 800 提高到 1500，避免目标 100 RPM、实际约 85～88 RPM 时输出被卡在约 900 PWM；总输出达到 `MOTOR_SPEED_PID_MAX_PWM=1600` 时，程序使用条件积分停止继续向饱和方向累积。
7. 左右轮为了达到相同 RPM 使用不同 PWM 是正常现象。例如左轮 910、右轮 890 只表示左右摩擦、负载或电机常数略有差异，不应强制两个 PWM 相等。

### 6. 最后决定是否需要 Kd

编码器在 20 ms 周期内得到的是离散计数，微分项容易放大量化噪声。通常速度环用 PI 就够了。

只有在 Kp、Ki 已经合理但阶跃超调仍明显时，才加入很小的 Kd。Kd 太大时，`left_pwm/right_pwm` 会随每个计数跳变产生尖峰。必要时可提高 `MOTOR_SPEED_PID_FILTER_KEEP` 来平滑速度，但数值越大，反馈延迟也越大。

### 7. 死区、目标与限幅

| 参数 | 作用 |
| --- | --- |
| `MOTOR_SPEED_PID_TARGET_RPM` | 固定模式目标，也是循环测试第一个目标 |
| `MOTOR_SPEED_PID_STEP_2_RPM` | 循环测试第二个目标，当前为 50 RPM |
| `MOTOR_SPEED_PID_STEP_3_RPM` | 循环测试第三个目标，当前为 120 RPM |
| `MOTOR_SPEED_PID_STEP_HOLD_MS` | 每个目标的保持时间，当前为 5000 ms |
| `MOTOR_SPEED_PID_ENABLE_TARGET_PROFILE` | 1：循环执行三段目标；0：只使用固定目标 |
| `MOTOR_SPEED_PID_FEEDFORWARD_PWM_PER_RPM` | 目标转速到基础 PWM 的前馈系数，当前为 8.0 |
| `MOTOR_SPEED_PID_DEAD_ZONE_PWM` | 克服电机低 PWM 不转的死区 |
| `MOTOR_SPEED_PID_MAX_PWM` | 单轮允许的最大 PWM |
| `MOTOR_SPEED_PID_FILTER_KEEP` | 速度低通滤波，越大越平滑、越慢 |
| `MOTOR_SPEED_PID_CONTROL_PERIOD_MS` | 速度计算与 PID 周期 |

死区应通过开环低 PWM 实验测出：从 0 缓慢提高 PWM，记录车轮刚能连续转动的数值，再略留余量。死区太大会造成低速跳动。

前馈只负责提前给出大致所需 PWM，闭环精度仍由 PI 保证。若启动或从 50 跳到
120 RPM 时出现明显超调，先把前馈系数从 8.0 降到 7.5；若仍然加速过慢且 PWM
没有碰到 1600 上限，可小步提高到 8.5。不要为了缩短上升时间直接大幅增加 Ki。

### 8. 从架空测试转到落地测试

架空调稳后，再把目标降到较低值并让车辆落地：

1. 观察负载增加后速度是否仍能回到目标。
2. 左右实际速度应接近，而左右 PWM 可以不同，这是 PID 正在补偿机械差异。
3. 分别轻触一个轮子制造短暂阻力，松手后速度应恢复且不持续振荡。
4. 更换不同电池电压重复测试，确认积分不会长时间饱和。
5. 再逐步提高目标 RPM，并为不同速度区间复查参数。

速度环合格标准是：左右轮都能快速接近目标，稳态误差小，负载扰动后可恢复，PWM 不长期饱和，且没有持续振荡。

## 六、推荐的调参记录表

每次实验至少记录：

| 项目 | 内容 |
| --- | --- |
| 测试名称 | 灰度 / 陀螺仪 / 速度环 |
| 固件参数 | Kp、Ki、Kd、基础速度、限幅、滤波 |
| 电池电压 | 测试开始时电压 |
| 场地与负载 | 架空/落地、地面材质、车上负载 |
| 上升或回正时间 | 从扰动到进入允许误差带的时间 |
| 最大超调 | 最大偏差或最高速度 |
| 稳态误差 | 稳定后的平均误差 |
| 是否振荡 | 振荡频率、幅度、是否衰减 |
| 串口日志文件 | 对应 CSV 文件名 |

不要同时修改多个参数。一次实验只验证一个假设，保留上一组可用参数，调坏时才能快速退回。

## 七、三套 PID 的关系

当前三个程序的目的都是“单独把一层调通”：

```text
灰度测试：灰度位置误差 -> 左右轮差速
陀螺仪测试：航向角 -> 目标角速度 -> 左右轮差速
速度测试：单轮目标 RPM -> 单轮 PWM
```

它们目前没有叠加运行。后续整车控制若把灰度或航向输出作为左右轮目标速度，再由速度环执行，应先保证速度内环明显快于灰度/航向外环，然后重新降低外环增益开始联调，不能直接把三套独立测试参数原样叠加。
